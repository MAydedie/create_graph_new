> 粒度：小模块级，附带文件路径与行号

## 一、系统输入（Entry Points）

### 1.1 用户输入（2 个）

| 输入 | 来源 | 说明 |
|------|------|------|
| **user_goal** | 前端输入框 / 快捷按钮 | 用户用自然语言描述的任务目标 |
| **workspace_root** | 前端工作区选择 | 目标项目根目录，如 `D:\catnet\CAT-Net-main\lib` |

### 1.2 入口代码位置

- **前端发起请求**：`ui/frontend/src/services/api.ts` 第 18-21 行  
  ```ts
  startChat: async (user_goal: string, workspace_root: string) => {
      const res = await axios.post(`${API_BASE}/chat`, { user_goal, workspace_root });
      return res.data;
  }
  ```
- **后端接收**：`ui/server.py` 第 254-261 行  
  ```python
  @app.post("/api/chat")
  async def start_chat(request: TaskRequest):
      task_id = task_manager.create_task(request)
      return {"task_id": task_id, "status": "started"}
  ```
- **TaskRequest 定义**：`ui/server.py` 第 50-53 行  
  ```python
  class TaskRequest(BaseModel):
      user_goal: str
      workspace_root: str
      restore_from: Optional[str] = None
  ```

## 二、整体数据流概览

```
[用户] user_goal + workspace_root
    │
    ▼
[API /api/chat] TaskRequest
    │
    ▼
[TaskManager.create_task] ──► 设置 WORKSPACE_ROOT 环境变量
    │                         创建 Agent 实例（Planner/Coder/Reviewer/ErrorSolver/Knowledge/Memory）
    │                         创建 Orchestrator
    │                         启动后台线程 _run_task_thread
    │
    ▼
[Orchestrator.execute_with_resolution_loop(user_goal)]
    │
    ├── _prepare_session_environment
    ├── _run_clarification
    ├── Planner.plan
    ├── for each step: _execute_step_with_resolution
    │       ├── _execute_step
    │       │     ├── Coder.execute_step (code_change/analysis)
    │       │     ├── Reviewer.review (changes or step)
    │       │     └── 或 Reviewer 直接执行 verify/run_tests
    │       └── 失败时: ErrorSolver.solve_error → _execute_fix_plan → 重试
    ├── _generate_final_summary
    └── force_summary_generation (finally)
    │
    ▼
[TaskManager] session.event_log.events ──► WebSocket 推送给前端
```

### 3.4 Orchestrator（协调者）

**文件**：`llm/agent/agents/orchestrator.py`

#### 3.4.1 主流程：execute_with_resolution_loop

| 步骤 | 输入 | 产出 | 代码位置 |
|------|------|------|----------|
| 1. 创建 Session | user_goal | TaskSession | 第 585-586 行 |
| 2. 准备环境 | session, user_goal | 注入 RAG/Context/FileStructure | 第 589 行 → _prepare_session_environment |
| 3. 澄清思考 | session, user_goal | thought 事件写入 event_log | 第 592 行 → _run_clarification |
| 4. 生成计划 | session, planner | plan 写入 session.plan，plan_created 事件 | 第 596-605 行 |
| 5. 执行循环 | steps | 每步 _execute_step_with_resolution | 第 619-646 行 |
| 6. 完成 | - | mark_completed，_generate_final_summary | 第 648-659 行 |
| 7. 异常 | Exception | mark_failed，force_summary_generation | 第 662-665 行 |

#### 3.4.2 _prepare_session_environment

| 子步骤 | 输入 | 产出 | 代码位置 |
|--------|------|------|----------|
| RAG 知识 | user_goal | session.rag_knowledge | 第 839-842 行 |
| 对话上下文 | user_goal | session.conversation_context | 第 844-849 行 |
| 文件扫描 | user_goal | session.file_structure | 第 851-854 行 |

#### 3.4.3 _scan_files_from_goal

| 输入 | 产出 | 代码位置 |
|------|------|----------|
| user_goal | `{scanned, paths, file_structure, source_files, test_files}` | 第 447-567 行 |

从 user_goal 中正则提取路径 → 遍历 .py 文件 → 分类 source_files / test_files → 构建 file_structure 文本。

#### 3.4.4 _run_clarification

| 输入 | 产出 | 代码位置 |
|------|------|----------|
| session, user_goal | thought 事件 (event_type="thought") | 第 856-871 行 |

调用 `_generate_clarification(user_goal)` 用 LLM 生成澄清文本，写入 event_log。

#### 3.4.5 _execute_step_with_resolution（死磕逻辑）

| 输入 | 产出 | 代码位置 |
|------|------|----------|
| session, step, step_id, all_errors, all_micro_plans | bool（是否成功/跳过） | 第 668-807 行 |

循环：`_execute_step` → 失败则 `get_last_error` → `_classify_error_type` → 若 assertion_error 且为 verify 则直接返回 False → 否则调用 ErrorSolver.solve_error → 若 give_up 则 skip_step 返回 False → 否则 `_execute_fix_plan` 执行 micro_plan → 继续重试。

#### 3.4.6 _execute_step（单步执行）

| 输入 | 产出 | 代码位置 |
|------|------|----------|
| session, step_id, step | bool | 第 294-389 行 |

- **code_change / analysis**：Coder.execute_step → Reviewer.review(result)；
- **verify**：直接 Reviewer.review(step)，内部对 run_tests 调用 _run_tests。

#### 3.4.7 _execute_fix_plan

| 输入 | 产出 | 代码位置 |
|------|------|----------|
| micro_plan | bool | 第 936-966 行 |

创建临时 fix_session，注入主 session 的 rag/conversation/file_structure，调用 `_execute_plan_v2` 执行 micro_plan 的 steps。

#### 3.4.8 _generate_final_summary / force_summary_generation

| 方法 | 输入 | 产出 | 代码位置 |

## 四、事件流（从前端视角）

1. 用户输入 goal → 调用 `POST /api/chat` → 返回 `task_id`
2. 前端连接 `WebSocket /ws/{task_id}`
3. 后端轮询 `task_manager.get_session(task_id).event_log.events`
4. 新事件通过 `ws.send_json({ type: "event", data: event.to_dict() })` 推送
5. 事件类型：`user_input`（前端自己加）、`thought`、`plan_created`、`step_start`、`step_complete`、`step_fail`、`step_skipped`、`error_resolution`、`final_summary`、`task_complete`
6. 任务结束时发送 `{ type: "status", data: "completed" | "failed" }`

## 五、关键数据流简图

```
user_goal, workspace_root
    │
    ├─► WORKSPACE_ROOT (env)
    │
    ├─► KnowledgeAgent.get_context_for_goal ──► session.rag_knowledge
    │
    ├─► MemoryAgent.get_relevant_context ──► session.conversation_context
    │
    ├─► _scan_files_from_goal(user_goal) ──► session.file_structure
    │
    ├─► _generate_clarification ──► event_log: thought
    │
    ├─► Planner.plan(user_goal, planner_context) ──► session.plan, event_log: plan_created
    │
    └─► for step in plan.steps:
            │
            ├─► [code_change/analysis] Coder.execute_step(step, coder_context)
            │       └─► result ──► Reviewer.review(result) ──► session.update_state(complete/fail)
            │
            ├─► [verify] Reviewer.review(step) ──► _run_tests(target) 或直接通过
            │       └─► session.update_state(complete/fail)
            │
            └─► [fail] ErrorSolver.solve_error ──► micro_plan ──► _execute_fix_plan
                    └─► 重试 _execute_step 或 skip_step
```

## 六、输出与最终结果

| 产出 | 位置 | 说明 |
|------|------|------|
| **task_id** | API 响应 | 供前端连接 WebSocket、展示进度 |
| **event_log.events** | TaskSession | 经 WebSocket 推送，前端渲染 ActiveTaskView |
| **final_summary** | Orchestrator | event_type="final_summary"，含 LLM 生成的任务总结 |
| **session.step_results** | TaskSession | 各步骤执行结果，供 _generate_final_summary 使用 |
| **session.plan** | TaskSession | 计划文件可保存到 workspace/.agent/plans/ |
