## 🚀 重新规划的路线图

### Phase 4: 实时反馈与事件流 (前端优先)
*替代原有的“REPL/命令行”阶段，以符合用户对“网页 UI”的偏好。*

**目标**：在现有的网页 UI 上提供 Agent “思维过程”的实时可见性，而无需使用命令行。

*   **4.1 后端事件流 (SSE/WebSocket)**
    *   [ ] 创建 `EventStreamer` 类，挂钩到 `Orchestrator` 和 `Agent` 的日志中。
    *   [ ] 在 Flask/FastAPI 后端实现服务器发送事件 (SSE) 端点。
    *   [ ] 广播事件：`agent_thought`（Agent 思考）、`tool_call`（工具调用）、`console_output`（控制台输出）、`error`（错误）。
*   **4.2 前端集成**
    *   [ ] 更新 React 前端以订阅事件流。
    *   [ ] 在 Web UI 中添加“实时 Agent 状态”终端视图（可折叠/悬浮）。
    *   [ ] 可视化“当前步骤”、“并行任务”和“错误”。

### Phase 5.1: 并行执行优化 (尝试性)
*增强 Phase 3.2，引入真正的依赖管理。*

**目标**：超越简单的“N 个一批”并行，实现真正的依赖图 (DAG)，为复杂的重构任务实现最大效率。

*   **5.1.1 依赖图逻辑**
    *   [ ] 增强 `TaskTool`，允许每个子任务接受 `dependencies`（依赖项）列表。
    *   [ ] 实现拓扑排序以确定执行顺序。
    *   [ ] 动态调度：一旦任务 A 完成，立即启动任务 B（如果 A 是 B 的唯一依赖）。
*   **5.1.2 资源锁定**
    *   [ ] 实现简单的文件级锁定，防止两个并行 Agent 编辑同一个文件。

### Phase 6: 系统提示词优化 (注入 "Claude Code" 的灵魂)
*用户请求的新阶段。*

**目标**：将 `claude-code` 中经过实战检验的高质量系统提示词注入到系统中，增强 Agent 的智能和角色感。

*   **6.1 提示词提取与移植**
    *   [ ] 从 `claude_code_reverse_implementation_plan.md` 和其他来源中提取 `PlannerAgent`、`CoderAgent`、`ReviewerAgent` 的提示词。
    *   [ ] 适配“思维链” (CoT) 和“角色扮演”指令。
    *   [ ] 使用“诊断型 Agent”模式优化 `ReflexionEngine` 的提示词。

    *   [ ] 重构 `prompts.py` 以支持模块化的提示词组合（基础角色 + 任务模式 + 约束条件）。
